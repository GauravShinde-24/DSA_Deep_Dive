
# 🚀 DSA Problem Solving & Algorithms

Welcome to my Data Structures and Algorithms repository! This space is dedicated to my journey of mastering foundational computer science concepts, optimizing code, and tackling algorithmic challenges.

## 🧠 My Approach

While the internet is full of standard solutions, the primary goal of this repository is to document **my unique logic and thought processes**. For every problem solved here, I focus on:
*   **The "Why" over the "What":** Breaking down why a specific data structure was chosen.
*   **End-to-End Problem Solving:** From the initial brute-force intuition to the final optimized solution.
*   **Custom Logic:** Highlighting my personal approach to edge cases and optimization bottlenecks.

## 🗂️ Topics Covered

This repository is organized by data structures and algorithmic paradigms. Here is what you will find inside:

### 🔹 Linear Data Structures
*   **[Arrays & Strings]** - Two-pointer techniques, sliding window, prefix sums.
*   **[Linked Lists]** - Singly linked lists, cycle detection, reversal algorithms.
*   **[Doubly Linked Lists]** - LRU cache implementations, bidirectional traversal.
*   **[Stacks & Queues]** - Monotonic stacks, expression parsing, BFS foundations.

### 🔹 Non-Linear Data Structures
*   **[Trees]** - Binary Trees, Binary Search Trees (BST), traversals (Inorder, Preorder, Postorder), Lowest Common Ancestor.
*   **[Heaps / Priority Queues]** - Top K elements, median in a data stream, heapify operations.

### 🔹 Algorithmic Concepts
*   **[Sorting Algorithms]** - Implementations of Merge Sort, Quick Sort, Heap Sort, and custom comparators.
*   **[Searching]** - Binary search on answer, rotated sorted arrays.

*(More topics like Graphs and Dynamic Programming will be added as the repository grows.)*

## 📂 Repository Structure

Each topic folder contains individual files for the problems. Inside each file, you can expect:
1.  **Problem Statement:** A brief description of the challenge.
2.  **Intuition:** The initial thought process.
3.  **Approach:** The logic used to solve the problem (Time & Space Complexity included).
4.  **Code:** Clean, well-documented code.
